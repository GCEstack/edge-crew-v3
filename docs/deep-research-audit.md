# Phase 1: Architecture Review

## 1.1 High-Level Architecture

```
┌─────────────────┐      ┌──────────────────────┐      ┌─────────────────────┐
│   Vite React    │◄────►│  Vercel Edge Routes  │◄────►│   Supabase Postgres │
│   (web/src)     │      │  /api/games          │      │   Auth / RLS        │
│   PWA SPA       │      │  /api/analyze        │      │   Realtime          │
└─────────────────┘      │  /api/jobs/:id       │      │   pg_cron           │
                         └──────────────────────┘      └─────────────────────┘
                                                                    ▲
                                                                    │
                                                         ┌────────┘
                                                         │  model_jobs / model_responses
                                               ┌─────────▼─────────┐
                                               │  AI Worker          │
                                               │  app/ai_worker.py   │
                                               │  (Fly.io container) │
                                               └─────────────────────┘
```

**Data Flow:**
1. **Slate fetch** — AI worker calls ESPN scoreboards for today + tomorrow and stores games in `games`.
2. **Analysis job** — User taps *Analyze* in the web app, which inserts a row into `model_jobs`.
3. **AI panel** — Worker runs a roster of models (Groq, xAI Grok, Perplexity, Moonshot Kimi, etc.) and a deterministic grade engine.
4. **Gatekeeper** — Moonshot Kimi acts as a final validation layer on non-fast runs.
5. **Results** — Grades, model breakdowns, picks, and convergence are written to `grades` and `model_responses`.
6. **Display** — Frontend reads from Vercel edge routes and renders the slate.

### Key FastAPI Endpoints
- `/health` — Comprehensive health check (DB, Redis, external APIs, disk, memory)

- `/docs`, `/redoc`, `/openapi.json` — **Disabled in production** (`DEBUG=false`)

### AI Worker (`app/ai_worker.py` — 419 lines)
- **Runtime:** Long-running Python container (Fly.io / Render / Railway)
- **Job Queue:** Polls Supabase `model_jobs` table every 5 seconds
- **Pipeline:**  
  `model_jobs` → `_analyze_games_impl()` → AI panel calls → convergence → write `model_responses` + `grades`
- **Heartbeat:** Writes to `worker_heartbeats` every 60 seconds
- **Kalshi Sync:** Syncs open sports markets every 3600 seconds (optional)
- **Timeout:** Per-job hard timeout 600 seconds

### Convergence Pipeline
1. **Deterministic Grade Engine** (`grade_engine.py`) — 50+ variables, pure math, no AI
2. **AI Panel** — Each model returns `grade`, `score`, `confidence`, `pick`, `thesis`, `key_factors`
3. **Bayesian Fusion** — Weight = confidence², produces consensus score
4. **Convergence Status:** `LOCK` / `ALIGNED` / `DIVERGENT` / `CONFLICT` / `CLOSE` / `SPLIT`
5. **Gatekeeper** — Moonshot Kimi reviews the full slate and returns top 1–3 plays + trap flags

### Prompt Engineering
- Each model has a **distinct persona** to fight groupthink (contrarian, data-driven, situational, etc.)
- Batch prompts for multi-game slates to reduce API call count
- Structured JSON output enforced with strict parsing and fallback

## 1.9 Security Architecture

### Authentication
- **Primary:** Supabase Auth (JWT-based)
- **Legacy:** FastAPI JWT middleware (`app/middleware/auth.py`) with HS256
- **PIN Login:** 4-digit PIN padded to 6 characters for Supabase Auth minimum
- **Crew Users:** `peter`, `chinny`, `jimmy` — default PIN `0000` (padded to `000000`)
- **Role-Based Access:** `require_role("admin")` factory for endpoint protection
- **Token Blacklisting:** Redis-backed SHA256 hash storage

### Authorization (RLS)
- **User-scoped:** `profiles`, `picks`, `locked_games`, `gut_picks` — `auth.uid() = user_id`
- **Public read:** `games`, `teams`, `kalshi_markets`, `sports_config`
- **Service role:** `grades`, `model_responses`, `model_jobs`, `odds_history`, `model_performance`

### Input Validation (`app/core/validation.py`)
- SQL injection pattern detection (27+ regex patterns)
- XSS sanitization (script tags, event handlers, `javascript:`, data URIs, HTML escaping)
- Pydantic `ValidatedRequest` base model with `extra="forbid"`
- String field auto-sanitization on all request models

### Security Headers (`app/middleware/headers.py`)
- `Strict-Transport-Security: max-age=31536000; includeSubDomains; preload`
- `X-Frame-Options: DENY`
- `X-Content-Type-Options: nosniff`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Permissions-Policy` (camera=(), microphone=(), geolocation=(), etc.)
- `Content-Security-Policy` (currently Report-Only mode, configurable via `CSP_REPORT_ONLY`)
- `X-Request-ID` request tracing
- `Cache-Control: no-store` for all `/api/` responses

### Rate Limiting (`app/middleware/rate_limit.py`)
- Redis-backed sliding window
- Default: 100 requests / 60 seconds
- Admin: 50 requests / 60 seconds
- Per-user (authenticated) or per-IP (anonymous, X-Forwarded-For aware)
- Graceful fallback when Redis unavailable
- Returns `X-RateLimit-*` headers and `Retry-After`

## 2.2 Security Holes

### ⚠️ CORS `*` in Vercel Edge Routes
- **Status:** `web/api/_shared.ts` line 7 sets `Access-Control-Allow-Origin: '*'`
- **Impact:** Edge API routes are exposed to any domain. Even though the main FastAPI CORS is strict, the Vercel routes bypass it.
- **Severity:** HIGH
- **File:** `web/api/_shared.ts` line 7

### ⚠️ CORS `*` in Supabase Edge Functions
- **Status:** `supabase/functions/grade/index.ts` line 8 sets `Access-Control-Allow-Origin: '*'`
- **Impact:** Grade engine edge function exposed to any domain
- **Severity:** HIGH
- **File:** `supabase/functions/grade/index.ts` line 8

### ⚠️ Weak Default Passwords
- **Status:** All crew accounts use PIN `0000` (padded to `000000`)
- **Impact:** Trivial brute-force. Anyone can guess the PIN in 10,000 attempts.
- **Severity:** CRITICAL
- **File:** `web/src/services/api.ts` lines 97–104, `README.md` line 230

### ⚠️ No Rate Limiting on Vercel Edge Routes
- **Status:** Vercel edge functions have no rate limiting middleware
- **Impact:** `/api/analyze` can be spammed to flood the AI worker queue. `/api/jobs/:id` can be polled aggressively.
- **Severity:** HIGH
- **File:** `web/api/analyze.ts`, `web/api/jobs.ts`

### ⚠️ No Input Validation on Edge Routes
- **Status:** Edge routes check sport validity but don't sanitize game_id, league, or other inputs
- **Impact:** Potential SQL injection via Supabase client if crafted maliciously (though Supabase JS client uses parameterized queries)
- **Severity:** MEDIUM
- **File:** `web/api/analyze.ts` lines 23–30

### ⚠️ Service Role Key Exposure Risk
- **Status:** `SUPABASE_SERVICE_ROLE_KEY` is used in Vercel Edge Routes (`web/api/_shared.ts` imports `supabase-server.ts` which uses service role)
- **Impact:** If Vercel project is compromised, service role key grants full database access bypassing RLS
- **Severity:** HIGH
- **File:** `web/src/lib/supabase-server.ts`

### ⚠️ JWT Secret in `.env.example`
- **Status:** `.env.example` contains a placeholder JWT secret `dev-secret-key-change-in-production` (24 chars, under 32)
- **Impact:** If someone copies `.env.example` directly to `.env` without changing, the JWT secret fails the Pydantic validation but the app might still start with a fallback
- **Severity:** MEDIUM
- **File:** `.env.example` line 24, `app/core/config.py` line 50

### ⚠️ No HTTPS Enforcement in Development
- **Status:** `localhost:3000` and `localhost:5173` are allowed in CORS without HTTPS
- **Impact:** Expected for dev, but `security_settings.debug` can be accidentally set to `true` in production
- **Severity:** LOW
- **File:** `app/core/config.py` lines 60–66

### ⚠️ No Request Size Limits
- **Status:** FastAPI default body size limits apply. No explicit `max_request_size` configured.
- **Impact:** Large payloads could cause memory exhaustion on the 512MB Render tier
- **Severity:** MEDIUM
- **File:** `app/main.py`

### ⚠️ No API Key for Internal Service Communication
- **Status:** `INTERNAL_API_KEY` is defined in `.env.example` but never validated in the middleware
- **Impact:** If the internal API key is meant to protect service-to-service calls, it's not actually used
- **Severity:** MEDIUM
- **File:** `.env.example` line 27, `app/middleware/auth.py` (no internal API key check)

### ⚠️ Missing Content Security Policy Nonce
- **Status:** CSP uses `'unsafe-inline'` and `'unsafe-eval'` for scripts
- **Impact:** XSS can still execute inline scripts despite CSP
- **Severity:** MEDIUM
- **File:** `app/middleware/headers.py` lines 63–73

### ⚠️ No HSTS on Vercel (Edge Routes)
- **Status:** HSTS is set by FastAPI middleware but NOT by Vercel Edge routes
- **Impact:** Vercel-served responses lack HSTS headers
- **Severity:** LOW
- **File:** `web/api/_shared.ts` (no HSTS header)

## 2.3 Scalability Limits

### ❌ AI Worker Single-Instance Bottleneck
- **Status:** Only ONE AI worker instance polls `model_jobs`
- **Impact:** If the worker crashes or is overloaded, the entire analysis pipeline stops. No horizontal scaling.
- **Severity:** CRITICAL
- **File:** `app/ai_worker.py`, `fly.toml`

### ❌ No Job Queue Partitioning
- **Status:** All jobs go to a single `model_jobs` table with no priority queue
- **Impact:** A batch of 50 games will block single-game analysis requests for hours
- **Severity:** HIGH
- **File:** `supabase/migrations/006_realtime_and_ai_jobs.sql`

### ❌ No Worker Auto-Scaling
- **Status:** Fly.io config is static: `shared-cpu-2x`, `2gb` memory
- **Impact:** Cannot handle burst loads. CPU-bound AI inference is limited to 2 shared cores.
- **Severity:** HIGH
- **File:** `fly.toml` lines 8–9

### ❌ Redis as Single Point of Failure
- **Status:** Redis is required for rate limiting, caching, and token blacklisting. If Redis goes down, rate limiting is disabled (not failed-closed).
- **Impact:** During Redis outage, the system has no rate limiting and no caching
- **Severity:** HIGH
- **File:** `app/middleware/rate_limit.py` lines 60–66, `app/middleware/auth.py` lines 48–53

### ❌ File Persistence Not Scalable
- **Status:** JSON file persistence at `/tmp/ec8` or `/data` is used for users and picks when DB is unavailable
- **Impact:** Cannot scale beyond one container. Data is lost on container restart if using `/tmp/ec8`.
- **Severity:** MEDIUM
- **File:** `app/main.py` lines 116–146

### ❌ Supabase Realtime Broadcast Limits
- **Status:** Supabase Realtime has default connection limits (varies by plan, typically 200–500 concurrent)
- **Impact:** If the app goes viral, Realtime subscriptions for `model_responses` and `model_jobs` may hit limits
- **Severity:** MEDIUM
- **File:** N/A (Supabase platform limit)

### ❌ No CDN for Static Assets
- **Status:** Static assets served from Vercel (has edge CDN) but the unified Docker image serves from FastAPI
- **Impact:** If using the unified Docker container on Render, static assets are served from the container, not a CDN
- **Severity:** LOW
- **File:** `Dockerfile` line 17

```python
# app/ai_worker.py — modified polling logic

import os
WORKER_ID = os.environ.get("FLY_ALLOC_ID", "worker-0")
WORKER_INDEX = int(WORKER_ID.split("-")[-1]) if "-" in WORKER_ID else 0
TOTAL_WORKERS = int(os.environ.get("TOTAL_WORKERS", "1"))

def _claim_job(supabase: Client) -> Optional[Dict]:
    """Claim a pending job using id-based partitioning to avoid collisions."""
    # Partition by job ID mod worker count
    response = supabase.rpc(
        "claim_job_for_worker",
        {"worker_index": WORKER_INDEX, "total_workers": TOTAL_WORKERS}
    ).execute()
    return response.data[0] if response.data else None
```

```sql
-- supabase/migrations/019_worker_partitioning.sql
CREATE OR REPLACE FUNCTION claim_job_for_worker(worker_index INT, total_workers INT)
RETURNS TABLE(id UUID, sport TEXT, game_id TEXT, fast BOOLEAN) AS $$
BEGIN
  RETURN QUERY
  UPDATE model_jobs
  SET status = 'processing', worker_id = worker_index::text, started_at = NOW()
  WHERE id IN (
    SELECT id FROM model_jobs
    WHERE status = 'pending'
    AND MOD(id::bigint, total_workers) = worker_index
    ORDER BY created_at
    LIMIT 1
    FOR UPDATE SKIP LOCKED
  )
  RETURNING model_jobs.id, model_jobs.sport, model_jobs.game_id, model_jobs.fast;
END;
$$ LANGUAGE plpgsql;
```

```typescript
// web/api/_shared.ts
import { kv } from '@vercel/kv';

export async function rateLimit(request: Request, maxRequests: number = 30, windowSeconds: number = 60): Promise<{ allowed: boolean; remaining: number }> {
  const ip = request.headers.get('x-forwarded-for')?.split(',')[0]?.trim() || 'unknown';
  const key = `rate_limit:${ip}:${new URL(request.url).pathname}`;
  
  const current = await kv.incr(key);
  if (current === 1) {
    await kv.expire(key, windowSeconds);
  }
  
  return { allowed: current <= maxRequests, remaining: Math.max(0, maxRequests - current) };
}
```

```python
# app/middleware/logging.py
import time
import logging
from starlette.middleware.base import BaseHTTPMiddleware
from app.middleware.headers import get_request_id, get_client_ip

logger = logging.getLogger("edge-crew-v3.access")

class RequestLoggingMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
        start = time.time()
        request_id = getattr(request.state, 'request_id', 'unknown')
        
        response = await call_next(request)
        
        duration = time.time() - start
        logger.info(
            "request",
            extra={
                "request_id": request_id,
                "method": request.method,
                "path": request.url.path,
                "status_code": response.status_code,
                "duration_ms": round(duration * 1000, 2),
                "client_ip": get_client_ip(request),
                "user_id": getattr(request.state, 'user_id', None),
            }
        )
        
        response.headers["X-Response-Time"] = f"{duration:.3f}s"
        return response
```

### 🔴 MEDIUM: Fix Job Recovery on Worker Crash

**Severity:** MEDIUM  
**Impact:** Jobs stuck in `processing` status if worker crashes mid-job  
**File:** `app/ai_worker.py`  
**Effort:** 4 hours

**Fix:** Add a heartbeat-based job timeout:

```sql
-- supabase/migrations/019_job_timeout_recovery.sql
CREATE OR REPLACE FUNCTION recover_stalled_jobs(timeout_seconds INT DEFAULT 600)
RETURNS INT AS $$
DECLARE
  recovered_count INT;
BEGIN
  UPDATE model_jobs
  SET status = 'pending', worker_id = NULL, started_at = NULL
  WHERE status = 'processing'
    AND started_at < NOW() - (timeout_seconds || ' seconds')::interval;
  
  GET DIAGNOSTICS recovered_count = ROW_COUNT;
  RETURN recovered_count;
END;
$$ LANGUAGE plpgsql;

-- Add to cron (runs every 10 minutes)
SELECT cron.schedule('recover-stalled-jobs', '*/10 * * * *', 'SELECT recover_stalled_jobs(600);');
```

# Phase 5: Comparative Analysis

## 5.1 vs. AI Voice Apps (ElevenLabs, Bland, Retell)

**Not applicable directly** — Edge Crew is a text-based sports betting intelligence platform, not a voice app. However, the **AI inference pipeline** can be compared:

| Dimension | Edge Crew | Best Practice (AI Voice) | Gap |
|-----------|-----------|--------------------------|-----|
| **Latency** | 500–800ms cold start (Azure) | <200ms streaming | ❌ No streaming. No warm-keepalive. |
| **Concurrency** | Single worker, sequential jobs | Horizontal scaling with auto-scaling | ❌ No horizontal scaling. |
| **Fallback** | 8+ providers with auto-fallback | Multi-provider with circuit breaker | ✅ Excellent fallback roster. |
| **Caching** | Redis + in-memory | Edge caching + CDN | ⚠️ Redis is good but no edge caching for AI responses. |
| **Cost Optimization** | No model selection optimization | Smart routing based on latency/cost | ❌ No cost-aware model selection. |
| **Observability** | Basic health checks | Per-request traces with latency breakdown | ❌ No distributed tracing. |

**Recommendation:** Adopt patterns from voice AI platforms:
- Add a **warm-keepalive** ping to Azure every 30 seconds to avoid cold starts
- Implement **streaming responses** for the AI pipeline (SSE or WebSocket)
- Add **cost-per-request tracking** and route to cheaper models for low-stakes analysis
- Add **predictive auto-scaling** based on queue depth (`model_jobs` pending count)
